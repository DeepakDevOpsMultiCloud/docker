What happens when you run docker run nginx?


docker run nginx
Docker CLI sends the request to the Docker Daemon.
The daemon checks if the nginx image exists locally.
If not, it pulls the image from Docker Hub.
Image layers are downloaded.
A writable layer is added on top of the image.
Linux namespaces are created.
Cgroups are applied to limit resources.
A network interface is attached.
The container filesystem is mounted.
ENTRYPOINT and CMD are executed.
The Nginx process starts.
The container is now running.

docker container states.

| State                | Description                                                                                  |
| -------------------- | -------------------------------------------------------------------------------------------- |
| **Created**          | Container has been created but the main process hasn't started yet.                          |
| **Running**          | The application's main process is executing.                                                 |
| **Paused**           | All processes are temporarily frozen.                                                        |
| **Exited (Stopped)** | The main process has stopped, but the container still exists.                                |
| **Restarting**       | Docker is stopping and starting the container again (or retrying based on a restart policy). |
| **Removed**          | The container has been deleted.                                                              |

13. Backup a Volume

Example:

docker run --rm \
-v postgres-data:/data \
-v $(pwd):/backup \
ubuntu \
tar czf /backup/postgres-backup.tar.gz /data

The backup archive is created on the host.

Restore
docker run --rm \
-v postgres-data:/data \
-v $(pwd):/backup \
ubuntu \
tar xzf /backup/postgres-backup.tar.gz -C /

Always validate restore procedures before relying on them.

docker0 = The Linux bridge interface that implements the default bridge network.

How does a container access the Internet?

Suppose Container 1 wants to access google.com.

The packet flows like this:

Container (172.17.0.2)
        │
        ▼
docker0 (172.17.0.1)   Docker NAT happens here
        │
        ▼
Host (192.168.1.10)
        │
        ▼
Router (Public IP: 49.204.100.50) Router NAT happens here
        │
        ▼
Internet

Docker uses NAT on the host, and then your router performs NAT again. To the internet, the request appears to come from the router's public IP.
